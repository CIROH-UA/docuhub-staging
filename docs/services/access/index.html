CIROH provides access to public cloud services, HPC, and on-premises infrastructure to support the research projects of CIROH's members and partners.

Public Cloud

Accessing Public Cloud Services

CIROH has partnered with AWS and Google Cloud to provide access to their cloud computing services.

Note: If using CIROH-2i2c services, please see the "Accessing CIROH JupyterHub" section below for additional steps.

Getting Started

1

Requesting Project Access

Primary Investigators (PIs) or Workshop Leads leading CIROH projects or workshops may use this form to request cloud computing resources on AWS or Google Cloud. Access is available to all consortium members and partners.

Cloud Infrastructure Request Form

Your Responsibilities

CIROH Consortium members or partners are responsible for:

  • ✓Management of CIROH subaccounts assigned to them
  • ✓Project-specific software and environment configuration
  • ✓Handling account creation and/or access for project contacts

Cost of Use

  • ★Use of CIROH JupyterHub is free for all consortium projects
  • $Individual projects are allotted $500 monthly for AWS and Google Cloud
  • ⚠Projects exceeding budget may request additional funds below
Request Additional Budget
Cloud Computing

Accessing CIROH-2i2c JupyterHub

In partnership with 2i2c, CIROH provides JupyterHub with both CPU and GPU capabilities.

Getting Started

1

Submit Infrastructure Request

Submit a GitHub template request detailing your project requirements and specifications.

Cloud Infrastructure Request Form
2

Request Individual Access

After your project has been approved, submit one of the following forms to get access to CIROH JupyterHub environments.

CPU Access Request FormGPU Access Request Form

Requesting Custom Images

To request custom images:

  1. Create an environment.yml file by exporting your conda environment.
  2. Fill out the CIROH-2i2c JupyterHub Software Install form.
JupyterHub Software Install Form
On-Premises HPC

Accessing On-Premises Infrastructure

CIROH operates an on-premises infrastructure that includes high-performance computing (HPC) resources and specialized software via the Pantarhei and Wukong systems.

Getting Started

1

Submit Project Access Request

Primary Investigators (PIs) leading CIROH projects may use this form to request CIROH on-premise resources for their teams, including Pantarhei and Wukong.

On-premises Infrastructure Request Form
2

Request Individual Access

Non-UA users should complete the VPN Access Request section of the form before proceeding to the On-Premise Access Request section.

On-Premise Access Request Form

Prerequisites

Before requesting individual access, ensure the On-Premises Infrastructure Request Form has been completed by your PI.
High-Performance Computing

NSF ACCESS Allocations through CIROH

In collaboration with NSF ACCESS Allocations, CIROH offers access to all the resources that are available through an allocation on the Resources. This guide will walk you through the steps necessary to gain access to any resources using your NSF ACCESS allocation.

Getting Started

1

Submit Infrastructure Request

The PI for your project must submit the Infrastructure Request Form below to request team-wide access to a particular resource on ACCESS Allocation.

Infrastructure Request Form
2

Register for ACCESS AllocationIf needed

If you don't already have an ACCESS Allocation account, register for one using the link below.

ACCESS Allocation New User Registration
3

Submit NSF ACCESS Resources Request

After your project has been approved, using your ACCESS Allocation ID, submit the NSF ACCESS Resources Request form for individual user accounts.

NSF ACCESS Resources Request Form

Need Help?

If you are unable to access the forms, please contact the CIROH team at ciroh-it-admin@ua.edu for assistance.

You're All Set!

Once you're granted access, you're ready to begin using the resources!

Visit Your ACCESS Resources Page

Requesting Infrastructure Support for Conferences and Programs

CIROH Project Leads can request computing resources for workshops using this process. Available infrastructure includes all resources from ACCESS Allocations, CIROH-2i2c JupyterHub, AWS, and Google Cloud - accessible to all consortium members and partners.

Getting Started

1

Submit Workshop Requirements
