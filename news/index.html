NGEN-DataStream

🎉Major milestone for the NextGen Research DataStream! Version v2.0.0 marks a significant reorganization with the ngen-datastream repository splitting into modular components:

DataStreamCLI and ForcingProcessor are now independent repositories. This structures the infrastructure for the NRDS in a more conventional way in order to facilitate CI/CD. Going forward, ngen-datastream repository will continue to hold NRDS infrastructure, deployment tooling, and documentation.

ForcingProcessor

ForcingProcessor version v1.1.1 launches as an independent repository!This repository has been created from the forcingprocessor module within the NextGen Research DataStream's repository ngen-datastream. All related tooling is planned to be removed from the ngen-datastream repository. Future development on forcingprocessor will occur only in this repository. blog

"Focusing on Streamflow Data: New Software for Camera-Based Hydrologic Modeling"

Camera-based monitoring offers a promising, non-contact alternative to or augmentation of traditional streamflow gages. However, broad use of camera-based streamflow monitoring has been limited by operational challenges including how to collect, store, manage, and process the large volume of image and video data produced by monitoring cameras. Read how researchers at Utah State University utilized CIROH cyberinfrastructure to advance camera-based hydrologic monitoring.
